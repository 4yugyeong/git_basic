TIL_211228

1) git 이란? 세계에서 가장 많이 쓰이는 버전 관리 도구!!!
2) github 란? git의 원격 저장소 + git으로 할 수 있는 커뮤니티 도구!!!
3) 버전관리란? 최종, 최최종, 최최최종 이렇게 파일명을 만들지 않고도
누가 언제 무슨 내용을 바꾸었는지 알 수 있게 해주는 도구!!!
4) git 초기화란? 컴퓨터에 있는 프로젝트를 git이 관리하는 프로젝트로 만들 수 있습니다. 앞으로 이건 
		git이 관리할꺼야! 라고 하는 것이 git 초기화이다!!!
5) 커밋이란? 파일 수정하고 git에 올릴때 어떤 내용을 바꾸었는지 핵심만 간결하게 꺼서 add 한다!!
	    현재 프로젝트의 상태를 찰칵 저장하는 것! 스냅샷!!!
6) 커밋내용이란? 커밋한 수정 내역으로써, 언제 누가 무슨 내용을 추가하고 삭제하였는지 를 알 수 있다.
7) push란? local repo에서 github에 내용을 공유하려고 올릴때 원하는 파일을 클릭하고 보낸다!
8) pull이란? puch 한 파일을 누군가 커밋하고 수정 또는 삭제 되어진 내용을 다시 내 컴퓨터로 옮긴다!
9) clone 이란? remote repo의 정보를 내 컴퓨터로 옮길때 소스코드를 통하여 옮겨서 정보를 얻는다! 
10) tracking이란? 원격 리포와 로컬 리포를 연결시켜서 반영할 수 있다! 로컬 리포가 원격 리포를 연결하는 것을 트레킹이라고 한다!
11) 앞으로 git을 통해 하고 싶은일? 
당장 닥친 팀 프로젝트를 깃을 통해 팀원들과 서로 협업하고 싶다!!!
